# Bybit Style Notes for PawSift

Captured: 2026-07-15

Source: https://www.bybit.com/en/

These notes translate the source's visual language into PawSift. They do not authorize copying Bybit trademarks, exchange content, token data, illustrations, or source assets.

## Measured Tokens

- Body canvas: `#000000`
- Primary action: `#ff9c2e`
- Primary action text: `#121214`
- Heading/data text: `#ffffff`
- Body font: `Inter, system-ui, -apple-system, "Segoe UI", Roboto, Ubuntu, sans-serif`
- Base body size: `14px`
- Homepage display heading: `48px / 48px`, weight `800`
- Primary button: `16px / 21px`, weight `600`, `11px 24px`
- Dense market row: `14px / 14px`, `10px 0`

## PawSift Translation

- Use a 56 px black command bar with the PawSift mark, service status, ruleset version, and machine-contract links.
- Use a near-black page canvas and charcoal tool surfaces with thin low-contrast borders.
- Keep the actual audit workflow in the first viewport: pet/product ticket at left, verdict and receipt at right.
- Use orange only for primary actions, focus, and the selected mode. Do not flood backgrounds with it.
- Use green, amber, and red only for semantic outcomes; always pair color with verdict text and an icon.
- Keep typography compact and data-led. Reserve large type for the verdict, not promotional copy.
- Keep radii at 8 px or less even where the reference uses pill controls.
- On mobile, preserve the command bar, then stack the editable ticket before the result; keep all tap targets at least 44 px.
